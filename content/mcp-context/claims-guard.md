# Claims Guard

## 출처/표현 안전 규칙
- 기사 출처 URL, 언론사명, 기자명은 본문에 직접 노출하지 않습니다.
- 글의 신뢰도를 위해 통계 자료 등 수치에 대한 자료들의 출처가 존재한다면 노출합니다.
- 특정 기사의 사실관계를 왜곡하거나 과장하지 않습니다.
- 기사 내용을 그대로 복제하지 않고, 핵심 이슈를 고객 맥락으로 재구성합니다.

## 사실성 규칙
- 수치/성과는 내부 검증된 것만 사용합니다.
- 불확실한 수치는 정성 표현으로 대체합니다.
- 규제/법령 관련 표현은 단정형을 피하고 보수적으로 작성합니다.

## 톤 규칙
- 비난/공포 조장형 문구 금지
- 경쟁사 직접 비교/비방 금지
- "~할 수 있습니다" 중심의 신중한 표현 사용

## 최종 점검 체크리스트
- 기자명/언론사명이 남아 있지 않은가?
- 과장된 수치 표현이 없는가?
- 문체가 `~습니다/~입니다`로 통일되어 있는가?
- 문제→대응→시사점 흐름이 유지되는가?
